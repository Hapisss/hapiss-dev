# PROJECT.md

# Project Overview

This project is a complete redesign and rebuild of my personal portfolio.

The goal is not to recreate the existing website, but to build a significantly better version in terms of design, user experience, maintainability, and technical quality.

The existing production website is available at:

https://hapiss.dev

The previous source code may also exist inside this workspace and should be treated as a reference only.

Neither the live website nor the previous implementation should be copied blindly.

Analyze them critically and propose better solutions whenever appropriate.

---

# Project Vision

This portfolio should communicate who I am as an engineer.

It should demonstrate not only what I have built, but also how I think, solve problems, and approach software development.

Visitors should leave with the impression that I am a frontend-focused engineer capable of building polished user experiences while having practical experience with backend development and production software.

The website should feel intentional, mature, and trustworthy.

---

# Technical Direction

The initial version may use hardcoded content.

However, the architecture should make future CMS integration straightforward.

Whenever practical:

* Separate content from presentation.
* Keep data structures reusable.
* Avoid tightly coupling UI to content.

Do not introduce a CMS unless explicitly requested.

# Non Goals

This project is not intended to:

* Follow every current design trend.
* Maximize animations.
* Impress through visual effects alone.
* Become an experimental playground.

Technology should support communication, not become the main attraction.

---

# Definition of Done

The project is considered successful when it:

* Feels premium and intentional.
* Reflects strong frontend engineering.
* Demonstrates practical software engineering knowledge.
* Is accessible and responsive.
* Performs well.
* Is maintainable.
* Is easy to extend.
* Is ready for future CMS integration.

---

# Project Principles

* Every feature should have a reason.
* Every animation should improve usability.
* Every component should be reusable.
* Every section should communicate a clear message.
* Every page should answer a user question.
* If something exists only because it looks cool, reconsider whether it belongs.
