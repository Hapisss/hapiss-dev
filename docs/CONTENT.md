# CONTENT.md

# Content Strategy

This document defines how the website should communicate.

Every section should reinforce the same message.

The goal is consistency.

---

# Personal Positioning

I am a frontend developer with production experience who enjoys building polished user experiences.

I also have practical experience working across the backend development lifecycle, including APIs, databases, deployment, and system architecture.

Frontend is my primary specialization.

Backend knowledge supports my ability to build complete products.

The website should reflect both without creating confusion.

---

# Core Message

Every page should reinforce this idea:

"I build thoughtful, production-ready web applications."

Not:

"I know lots of frameworks."

Technology is evidence.

Engineering mindset is the message.

---

# Personality

The website should communicate:

* Professional
* Curious
* Reliable
* Detail-oriented
* Practical
* Calm
* Honest

Avoid sounding:

* Arrogant
* Overconfident
* Salesy
* Corporate
* Generic

---

# Writing Style

Use simple English.

Short sentences.

Avoid buzzwords.

Avoid marketing language.

Avoid exaggerated claims.

Show confidence through evidence instead of adjectives.

Prefer:

"I built..."

instead of

"I successfully delivered cutting-edge scalable enterprise solutions..."

---

# What Should Visitors Remember?

After leaving the website, visitors should remember:

* I specialize in frontend development.
* I have experience building production software.
* I care about quality.
* I understand software engineering beyond the UI.
* I pay attention to details.

---

---

Projects

Contact

Answer:

How can someone reach me?

---

# Storytelling Principles

Projects should tell stories.

Instead of:

"I built an admin dashboard."

Explain:

* What problem existed?
* Why was it difficult?
* What solution was implemented?
* What impact did it have?

People remember stories more than features.

---

# Things To Avoid

Avoid:

* Buzzwords
* Empty claims
* Generic developer clichés
* Long paragraphs
* Walls of text
* Technology dumping

Every sentence should provide value.

---

# Copywriting Rule

Do not write to impress.

Write to communicate.

If a sentence sounds impressive but adds no information, remove it.

Clarity always wins.
