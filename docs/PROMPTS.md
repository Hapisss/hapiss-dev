# PROMPTS.md

## Analyze Project

Read:

* AGENTS.md
* PROJECT.md
* CONTENT.md
* DESIGN.md
* TECH.md
* REFERENCES.md

Analyze the project and identify opportunities for improvement.
Do not write code.

---

## Architecture Review

Review the project architecture.

Identify technical debt.

Suggest improvements.

Wait for approval.

---

## UI Review

Review the current UI.

Evaluate:

* Visual hierarchy
* Spacing
* Typography
* Accessibility
* Consistency

Do not implement changes yet.

---

## Feature Implementation

Read all documentation first.

Create an implementation plan.

Wait for approval.

Implement only after approval.

---

## Final Review

Review the project as if you were:

* a frontend tech lead
* a recruiter
* a product designer

Provide actionable feedback and prioritize improvements.
